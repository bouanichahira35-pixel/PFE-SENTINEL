OBJECTIF :
Ne pas refaire le module Inventaire côté magasinier. La page actuelle /magasinier/inventaire doit rester simple et stable.

CONTEXTE :
La partie inventaire est déjà traitée dans le rapport avec le workflow suivant :
- le responsable lance une session d’inventaire ;
- le magasinier consulte les missions affectées ;
- le magasinier réalise le comptage terrain ;
- l’inventaire soumis devient visible côté responsable pour validation ou recomptage ;
- l’historique reste consultable en lecture seule côté magasinier.

MODIFICATIONS AUTORISÉES UNIQUEMENT :
1. Garder la page /magasinier/inventaire avec les sections actuelles :
   - “Comptage terrain”
   - “Mes missions”
   - “Historique (lecture seule)”

2. Ne pas ajouter de nouveaux modules lourds.
3. Ne pas ajouter de nouvelle logique complexe.
4. Ne pas créer de faux workflow non présent dans le backend.
5. Ne pas modifier le rôle du magasinier.
6. Ne pas donner au magasinier le droit de valider les écarts définitivement.
7. Ne pas déplacer l’inventaire vers une autre page.
8. Ne pas changer le sidebar.

AMÉLIORATIONS LÉGÈRES AUTORISÉES :
1. Améliorer seulement les textes affichés dans les états vides.
   Exemple pour “Mes missions” :
   - Titre : “Aucune mission active”
   - Texte : “Vous n’avez aucun inventaire à traiter pour le moment.”
   - Bouton : “Actualiser”

2. Améliorer seulement la présentation visuelle :
   - garder les cartes arrondies ;
   - garder les couleurs douces ;
   - éviter les grands tableaux vides ;
   - garder une hauteur raisonnable ;
   - conserver un style cohérent avec les autres pages magasinier.

3. Si une mission existe, afficher les colonnes déjà prévues :
   - Référence
   - Type
   - Magasin
   - Zone / famille / catégorie
   - Date prévue
   - Statut
   - Progression
   - Action

4. Dans l’historique, afficher uniquement les inventaires soumis ou terminés.
   Cette section doit rester en lecture seule.

RÈGLES MÉTIER À RESPECTER :
1. Le magasinier ne lance pas la validation finale.
2. Le magasinier ne valide pas les ajustements de stock.
3. Le magasinier ne décide pas du recomptage.
4. Le magasinier consulte ses missions, réalise le comptage et soumet le résultat.
5. Le responsable reste responsable de la validation finale ou de la demande de recomptage.

TEXTES À UTILISER :
- Titre de page : “Mes inventaires”
- Section 1 : “Comptage terrain”
- Section 2 : “Mes missions”
- Section 3 : “Historique (lecture seule)”
- État vide missions : “Aucune mission active”
- Sous-texte : “Vous n’avez aucun inventaire à traiter pour le moment.”
- État vide historique : “Aucun inventaire soumis ou terminé.”

CONTRAINTES STRICTES :
- Ne pas casser les routes existantes.
- Ne pas modifier les API existantes sauf correction mineure nécessaire.
- Ne pas ajouter de fausses données permanentes.
- Ne pas supprimer l’historique.
- Ne pas supprimer la section “Comptage terrain”.
- Ne pas transformer cette page en dashboard BI.
- Ne pas ajouter de fonctions non demandées.

BUT FINAL :
Conserver une page inventaire simple, propre et défendable devant le jury, compatible avec le rapport et centrée sur le rôle réel du magasinier.
