# Guide PFE (version simple)

Ce document explique l’application **SENTINEL** avec des mots faciles.

## 1) Idée générale
SENTINEL aide l’entreprise à **demander**, **valider**, **préparer** et **suivre** les ressources internes.  
Le système donne aussi des **alertes** et des **conseils IA** pour éviter les ruptures.

## 2) Les acteurs (très simple)
- **Demandeur** : demande un produit.
- **Magasinier** : prépare et sort les produits.
- **Responsable** : valide, surveille, décide.
- **Admin IT** : surveille le système, sécurité, incidents.
- **Fournisseur** : reçoit un email si commande (optionnel).

## 3) Le chemin “classique”
1. Demandeur crée une demande.  
2. Responsable valide ou refuse.  
3. Magasinier prépare et sort le stock.  
4. Le système garde l’historique + notifications.

## 4) Où est l’IA ?
- **Responsable** : alertes, priorités, mini‑rapports.  
- **Magasinier** : aide terrain (anomalies, conseils).  
- **Admin IT** : supervision technique de l’IA (pas de décisions).

## 5) Pourquoi c’est pro ?
- Traçabilité (qui fait quoi, quand).
- Rôles séparés (métier ≠ technique).
- Notifications et alertes.
- IA utilisée pour décider, pas pour remplacer l’humain.
